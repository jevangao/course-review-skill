# 课程课件复习资料生成 Skill 中文说明

本文件是 `course-slide-study-guide` 的中文配套说明，供维护者与使用者审阅。实际触发入口仍为仓库根目录的 `SKILL.md`。

## 功能定位

本 skill 用于从课程、讲座、工作坊或培训幻灯片中生成可核验的学习资料，包括：

- 详细 Markdown 复习讲义；
- 考前速查表；
- 公式表、术语表、方法卡片；
- 自测题、闪卡提示和练习清单；
- 课件覆盖范围与核验说明。

适用输入包括 `.pptx`、`.ppt`、PDF 版幻灯片，以及能够授权导出到本地的在线幻灯片。

## 核心原则

1. 跟随用户指定的语言、学习目标、详略程度和输出格式。
2. 默认只读处理来源课件，不改动原文件。
3. 大规模整理任务默认保留证据目录，包括清单、逐页文本、覆盖报告和视觉核验材料。
4. 输出资料必须说明覆盖了哪些文件、多少页以及存在何种限制。
5. 当内容存在差异时，显式区分：
   - `课件口径`：课件所教的定义、符号、条件或术语。
   - `通用补充`：为了理解而加入的标准背景知识。
   - `待核验`：疑似笔误、表述冲突、图示歧义或渲染不完整内容。
6. 对依赖视觉表达的重要内容，不得只根据抽取文字自行推断。

## 输入路线

| 输入类型 | 文本与结构处理 | 视觉核验 |
| --- | --- | --- |
| `.pptx` | 运行内置清点脚本和 PPTX 语料抽取脚本 | 逐页渲染或生成联系表 |
| PDF 幻灯片 | 使用可用的 PDF 提取工具，保留页码对应 | 渲染 PDF 页面并抽查重点页 |
| 旧版 `.ppt` | 在可用时转换为 `.pptx` 或 PDF；无法转换则报告未覆盖 | 核查转换后的页面 |
| 在线幻灯片 | 经授权导出稳定本地副本，并记录来源 | 检查导出版本 |

混合输入时，应形成统一的来源覆盖索引，并明确哪些输入使用了不同处理路线。

## 建议证据目录

```text
<输出根目录>/
├── evidence/
│   ├── source-inventory.json
│   ├── source-inventory.md
│   ├── pptx-corpus.json
│   ├── pptx-corpus.md
│   ├── coverage-report.md
│   └── visuals/<来源>/contact-sheet.png
└── <用户需要的学习资料>.md
```

对于多份课件或整学期课程，应默认保留证据目录，方便用户追查总结来源。

## 处理流程

### 1. 确定输出目标

当请求已经足够明确时，采用以下默认值直接执行：

- 语言跟随用户语言。
- 默认输出详细 Markdown 复习资料和简短核验说明。
- 默认面向正在复习所给课件的学习者。
- 默认进行结构化综合，而不是逐页摘抄。

仅当学习目标会显著改变产物时才追问，例如“考前冲刺资料”与“授课教案”的结构不同。

### 2. 清点全部来源

```bash
python3 scripts/inventory_slides.py --output-dir <证据目录> <文件或目录> [...]
```

该脚本会递归识别 `.pptx`、`.ppt` 和 `.pdf`，并统计 PPTX 页数。课件顺序不能只凭文件系统默认排序判断，应结合文件名、标题和章节内容推断，并在不确定时标注。

### 3. 提取 PPTX 可搜索语料

```bash
python3 scripts/extract_pptx_corpus.py --output-dir <证据目录> <PPTX 文件或目录> [...]
```

脚本会：

- 依据 OOXML 关系文件读取真实演示顺序；
- 按页提取文字；
- 仅提取备注正文，排除页码等备注占位符；
- 输出 Markdown 与 JSON 格式语料；
- 标记含图片、图表、连接线或无文字的视觉核验页。

提取语料用于检索和系统阅读，但不能替代对关键图解页面的查看。

### 4. 渲染与视觉检查

对于大型课件集合，应尽可能为每份课件生成逐页预览与联系表。重点查看：

- 算法推演、流程图、结构示意图和时间线；
- 数学公式、图表、数据表、几何图形；
- 代码执行过程、操作演示、实验配置和截图；
- 习题、小结、学习目标和考点提示页；
- 抽取脚本标记为需要视觉核验的页面。

若嵌入对象无法渲染、文字抽取明显缺失或存在课件与常识冲突，应写入核验说明，不得悄悄修正或忽略。

### 5. 建立知识模型

正式写作前，应整理：

- 章节顺序和主题地图；
- 学习目标与反复出现的重点；
- 定义、公式、流程、案例、算法或方法；
- 本课程特有的符号、命名和条件；
- 疑似错误、歧义内容与需要提醒学习者的位置。

最终资料应围绕学习任务重新组织内容，而不是简单连接幻灯片条目。

### 6. 生成学习资料

常见输出包括：

| 使用目的 | 主要产物 | 可增加内容 |
| --- | --- | --- |
| 系统学习一门课程 | 按依赖顺序组织的详细讲义 | 概念图、术语表、自测清单 |
| 考试复习 | 强调题型、方法与易错点的压缩资料 | 公式卡、最后检查清单 |
| 培训复盘 | 行动导向的手册 | 流程表、角色职责、术语说明 |
| 教学备课 | 教学提纲 | 目标、例题、提问建议 |
| 生成题库 | 知识目标索引 | 按模块和难度标记的题目 |

详细讲义至少建议包含：

```markdown
# <课程名称>复习资料

## 来源与覆盖范围
## 使用方法
## 课程知识地图
## 分模块知识整理
## 公式、流程或方法模板
## 比较与选择表
## 课件口径、注意事项与疑似错误
## 自测或练习检查单
## 课件覆盖索引
```

### 7. 进行覆盖与准确性校验

对于 PPTX 输入，运行：

```bash
python3 scripts/check_coverage.py \
  --inventory <证据目录>/source-inventory.json \
  --corpus <证据目录>/pptx-corpus.json \
  --output <证据目录>/coverage-report.md
```

此外还需人工确认：

- 用户期望处理的来源均已纳入，或已列明无法处理的输入；
- 支撑重要结论的视觉页面确实查看过；
- 公式、数字例题、符号和适用条件与课件一致；
- 通用补充和纠错意见没有伪装成课件原话；
- 输出 Markdown 可正常阅读，文件与链接可定位。

## 内置脚本

| 脚本 | 作用 |
| --- | --- |
| `scripts/inventory_slides.py` | 清点课件输入并统计 PPTX 页数 |
| `scripts/extract_pptx_corpus.py` | 提取页级文本、真实备注、视觉标记和 JSON/Markdown 语料 |
| `scripts/check_coverage.py` | 校验清点与抽取覆盖一致性并生成报告 |

## 适应不同学科的侧重点

| 学科/课件类型 | 资料整理重点 | 重点视觉检查 |
| --- | --- | --- |
| 算法与计算机 | 不变量、伪代码、复杂度、边界条件 | 执行过程、结构图、代码与表格 |
| 数学与统计 | 定义、条件、定理、证明思路、例题模式 | 公式、曲线、几何和数据图 |
| 理工与实验 | 模型、单位、装置、因果关系、计算 | 原理图、实验图表与公式注记 |
| 法律与管理 | 概念区分、制度框架、案例与决策条件 | 时间线、矩阵与条文图表 |
| 语言与人文 | 词汇、主题、语法或论证模式、例文 | 注释文本、地图与年代 |
| 产品与职业培训 | 操作流程、职责、工具、场景与合规事项 | 界面截图、业务流程和看板 |

## 维护说明

- `SKILL.md` 是供 Codex 触发和执行的正式文件，应保持简明且稳定。
- `references/` 保存仅在需要时加载的详细规则，避免主 skill 过长。
- `scripts/` 中的处理脚本应保持只读来源、确定性输出并通过真实样本回归。
- 中文说明用于维护和审阅；当正式流程变化时，应同步更新本文件。
